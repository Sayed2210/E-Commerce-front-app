## Required Setup

Create `.env` before running anything:

```env
NUXT_PUBLIC_API_BASE_URL=http://localhost:3000
NUXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_test_...  # optional, leave empty to disable
```

## Dev Commands

```bash
npm run dev        # http://localhost:3000
npm run build      # production build
npm run preview    # preview build
npm test           # vitest
npm run test:watch
npm run test:coverage
npx vitest tests/stores/auth.test.ts   # single test file
npm run lint
npm run lint:fix
npm run format
npm run tokens:sync   # sync design tokens from Stitch system → main.css + designTokens.ts
npm run tokens:check  # CI: fail if tokens drifted
```

## CI Pipeline Order

```
tokens:check → lint → test → build
```

## Architecture

**Nuxt 4** with `app/` directory convention.

### Path aliases

`~` and `@` resolve to `app/`.

### Component Naming (critical)

`nuxt.config.ts:55-58` sets `pathPrefix: false` — **every `.vue` filename under `app/components/` must be globally unique**. Subfolder names are organizational only. Two files with the same name will silently shadow each other at runtime.

### CSS / Design Tokens

`app/styles/main.css` contains Tailwind v4 `@theme` with design tokens. This file is **auto-generated by `scripts/sync-tokens.js`** — never edit manually. Run `npm run tokens:sync` after design token changes.

## Auth Layers

1. `app/utils/token.ts` — raw cookie read/write (SSR-safe)
2. `app/stores/auth.ts` — Pinia store
3. `app/composables/useAuth.ts` — login/register/logout/fetchUser
4. `app/utils/api.ts` — `useApiClient()` for authenticated API calls with auto-refresh
5. `app/middleware/` — `auth.ts`, `admin.ts`, `guest.ts`

Admin uses same endpoints as user — role check in `useAuth.ts` (line 20-23).
